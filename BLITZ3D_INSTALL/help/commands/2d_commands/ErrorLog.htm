ErrorLog

Parameters

None.

Description

When specific Blitz3D functions are run (and debug mode is disabled), these functions will check to make sure the information they are trying to parse is valid.
If the information is invalid, instead of continuing on or raising a runtime error, the function will be ended and the error message will be pushed back into an error log.

Using ErrorLog(), you can retrieve the error that has occured, as well as any furthur errors.

Fun fact: this function is where the "MAV-Less" in "Blitz3D MAV-Less" comes from.

Example

;There was a problem - notify the user
Text 5,5,"An error was caught! Details: "+ErrorLog()
