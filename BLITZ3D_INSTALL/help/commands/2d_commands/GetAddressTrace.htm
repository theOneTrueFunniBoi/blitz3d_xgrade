GetAddressTrace

Parameters

None.

Description

When specific Blitz3D functions are run (and debug mode is disabled), these functions will check to make sure the information they are trying to parse is valid.
If the information is invalid, instead of continuing on or raising a runtime error, the function will be ended and a memory address trace of the error will be pushed into a log.

Using GetAddressTrace(), you can retrieve the most recent stack trace.

Example

;There was a problem - notify the user
Text 5,5,"An error was caught! Memory: "+GetAddressTrace()
