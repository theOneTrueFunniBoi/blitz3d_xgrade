RuntimeError message$,[type]

Parameters

message$ - any valid string
type - optional type of runtime error to generate (integer value)

Description

When doing your own error trapping, use this command to pop up a fatal error and close the program. You can specify the error message that is displayed.

If specified, the following are the runtime error types controled by [type]:

• Default

Generates a runtime error using the standard type. The message$ string is passed in, followed by the generated stack and memory trace.

• '2'

Generates a runtime error using a non-standard type that does not contain a stack or memory trace.

• '3'

Generates a runtime error using the old type, as seen in non-mavless Blitz3D SoLoud versions.

Example

;There was a problem - raise an error and quit

Local whatHappened$ = "Missing required items: frogs.mp3"
RuntimeError("Installation corrupted! Error - "+whatHappened+" Please reinstall.",1)
